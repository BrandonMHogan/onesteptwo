<objective>
Wire the Go backend to PostgreSQL and implement the consent-gate endpoint. First establish the API contract by adding three endpoints and their schemas to `api/openapi.yaml` and regenerating `generated.go`, then add a `*sql.DB` to the `Server` struct and open the connection in `main.go`. Finally implement `POST /v1/children` as a single atomic transaction (D-07): insert the `consent_events` row, capture its generated id, then insert the `children` row referencing it — so no orphaned consent record and no child without consent is ever possible.

Purpose: REQ-009/REQ-010 (consent recorded before child, atomic ordering), REQ-C-001 (COPPA parental consent gate), REQ-013 (deletion endpoints declared in the contract), REQ-C-009 (no IP in consent record).
Output: Updated OpenAPI spec + regenerated interface, a DB-backed Server, and a tested `PostV1Children` handler.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-2-01 | Tampering | consent gate bypass (child without consent) | mitigate | Single BeginTx inserts consent_events then children with the returned id; DB FK (Plan 01) is the backstop |
| T-2-03 | Information Disclosure | IP captured into consent_events | mitigate | Handler reads no r.RemoteAddr/X-Forwarded-For; consent INSERT column list fixed to D-06 four fields (REQ-C-009) |
| T-2-05 | Tampering | SQL injection via header/body values | mitigate | All queries parameterized with `$N`; acceptance criteria forbid fmt.Sprintf SQL |
| T-2-06 | Elevation of Privilege | endpoint reachable without auth in production | mitigate (Phase 2) | Placeholder header check returns 401 when identity absent; `// TODO Phase 3` marks JWT replacement; endpoints MUST NOT ship to production before Phase 3 auth middleware |
