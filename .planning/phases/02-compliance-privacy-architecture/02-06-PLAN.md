| Handler → Clerk API | Org member list (read, pre-tx) and Organization deletion (write, post-commit) cross the network boundary |
| Handler → PostgreSQL | Multi-statement hard-delete cascade + org-wide device_tokens delete + audit, all in one transaction |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-2-02 | Tampering / Elevation | erasing another org's account (no JWT org check in Phase 2) | accept (Phase 2) → mitigate (Phase 3) | Placeholder header gives 401 when absent; existing `// TODO Phase 3` org-ownership check stays. Endpoints MUST NOT ship to production before Phase 3 |
| T-2-14 | Tampering | partial erasure if Clerk org deletion is run inside the tx and fails | mitigate | Member-list fetch happens before BeginTx; Clerk org deletion happens only after tx.Commit (Pitfall 2); a Clerk failure is logged, not rolled back, because DB data is already gone |
| T-2-15 | Information Disclosure | leaking Clerk API error bodies to clients | mitigate | clerk.go returns status-coded errors without response bodies; handler maps them to a generic RFC 7807 500 detail |
| T-2-11 | Repudiation | account-deletion audit row untraceable to the org (was gen_random_uuid) | mitigate | target_id is a deterministic UUIDv5 of clerk_org_id — recomputable to locate an org's audit rows (REQ-C-008) |
| T-2-05 | Tampering | SQL injection via headers / member ids | mitigate | All deletes parameterized; member ids bound via pq.Array($1) |
| T-2-SC | Tampering | supply-chain (package installs) | accept | No new modules added — uses stdlib net/http + already-present github.com/lib/pq and github.com/google/uuid (go mod tidy promotes uuid from indirect to direct; no download) |
</threat_model>
