<objective>
Create the goose SQL migration `00002_schema.sql` that defines the entire server-side relational schema for OneStepTwo, and reconcile `docs/04-data-model.md` with the locked decisions D-01 (event_type enum values) and D-05 (FK direction flip). This migration is the database-level enforcement layer for the consent gate and data minimisation: the `children.consent_event_id NOT NULL REFERENCES consent_events(id)` constraint makes it physically impossible to insert a child profile without a pre-existing consent record.

Purpose: Lock all compliance obligations into the schema before any child data can flow (REQ-008 data minimisation, REQ-C-005 PII-in-Clerk-only, REQ-C-009 no-IP, REQ-NF-003 no auto-expiry).
Output: `backend/db/migrations/00002_schema.sql` (six tables, two enums, FK-dependency-ordered Up and reverse-ordered Down) and an updated `docs/04-data-model.md`.
</objective>

No FK to any other table (must survive after data deletion).
