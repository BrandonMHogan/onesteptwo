<objective>
Implement the two GDPR/COPPA erasure endpoints, replacing the Plan 02 stub bodies. `DELETE /v1/children/{id}` hard-deletes a child's data in FK-safe order (the FK direction from D-05 means the children row must be deleted BEFORE its consent_events row), records an audit row, and returns the D-10 confirmation. `DELETE /v1/account` cascades child deletion across every child in the requesting org and records an account-level audit row. Each handler first sweeps `erasure_audit` rows older than 90 days (D-12 sweep-on-request) inside the same transaction.

Purpose: REQ-011 (child deletion cascade), REQ-012 (account deletion cascade), REQ-013 (verified deletion endpoints), REQ-014 + REQ-C-008 (erasure right, audit, 90-day purge), REQ-C-002/003/004 (GDPR/PIPEDA/POPIA erasure obligations satisfied by the same mechanism).
Output: Two tested erasure handlers in server.go and an httptest file covering their auth/validation paths and response shape.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-2-02 | Tampering / Elevation | erasing another org's child (IDOR — no org check in Phase 2) | accept (Phase 2) → mitigate (Phase 3) | Placeholder header gives a 401 when absent; `// TODO Phase 3: verify children.clerk_org_id matches requester org` marks the JWT org check that closes the gap. Endpoints MUST NOT ship to production before Phase 3 |
| T-2-04 | Repudiation | deletion with no audit trail | mitigate | erasure_audit INSERT runs inside the same tx as the deletes; tx rollback removes both — no deletion without an audit row, no audit without a deletion |
| T-2-10 | Tampering | FK-violation leaving partially-deleted/inconsistent state | mitigate | FK-safe order (children before consent_events); every ExecContext error checked before Commit; defer Rollback on any failure |
| T-2-05 | Tampering | SQL injection via path id / headers | mitigate | All deletes parameterized with `$N` |
| T-2-11 | Repudiation | audit table growing unbounded / stale-data retention | mitigate | D-12 sweep-on-request purges erasure_audit rows >90 days at the start of every deletion tx (REQ-C-008) |
| T-2-SC | Tampering | supply-chain (package installs) | accept | No new packages introduced |
