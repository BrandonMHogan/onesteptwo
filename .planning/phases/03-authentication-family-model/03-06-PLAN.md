<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user → ClerkKit | Credentials, sessions, invitations handled by Clerk |
| admin → invitation | Only org:admin may invite; enforced client-side (UX) + server-side (03-01) |
| caregiver session → API | Caregiver JWT carries org:caregiver; server rejects admin-only ops |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-05 | Elevation of Privilege | invite visible to caregivers on iOS | mitigate | InviteCaregiverView gated on `org:admin`; server enforces HasRole (REQ-016) |
| T-3-13 | Tampering | invitation to a non-existent role | mitigate | `org:caregiver` role pre-created in Clerk (03-04 Task 1); exact key used |
| T-3-14 | Spoofing | stale org context after switching | mitigate | `setActive` reissues the JWT with the new org_id |
| T-3-15 | Information Disclosure | invitation error leaking PII | accept | Fixed UI strings; raw ClerkKit errors not surfaced |
</threat_model>
