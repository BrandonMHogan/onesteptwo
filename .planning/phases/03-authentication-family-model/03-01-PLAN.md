<objective>
Replace the Phase 2 placeholder header-based identity (`X-Clerk-User-Id` / `X-Clerk-Org-Id`) in the Go API with real Clerk JWT validation and per-handler authorization. Wire the Clerk Go SDK v2 middleware (azp validation + clock-skew leeway) in `main.go`, extract session claims in every protected handler, enforce active-organization context and the `org:admin` role, and close the known IDOR gap (T-2-02) in `DeleteV1ChildrenId`.

Purpose: This is the server-side security boundary for the entire product. Every child-data operation must be gated by a cryptographically verified Clerk JWT with org context and role enforced server-side — clients cannot be trusted to assert identity.

Output: clerk-sdk-go/v2 in go.mod; authMiddleware wrapping the mux; claims-based auth + org + role checks in all three handlers; IDOR ownership check; a green `auth_test.go` plus migrated existing handler tests.
</objective>
