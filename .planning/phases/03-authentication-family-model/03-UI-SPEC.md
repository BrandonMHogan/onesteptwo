# Phase 3 — UI Design Contract

> Visual and interaction contract for the authentication and family model phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Platform note:** This is a native mobile project (Android + iOS). shadcn and Tailwind are not applicable. Design tokens map to Jetpack Compose `MaterialTheme` on Android and SwiftUI system components on iOS. Phase 3 builds functional authentication scaffolding. Phase 4 (UI/UX Design) is the canonical design system authority and will override the Phase 3 working colors. Typography and spacing established here should be treated as durable defaults unless Phase 4 explicitly overrides them.

## Spacing Scale

All values in `dp` (Android) / `pt` (iOS). Values are multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4dp | Icon gaps, inline text padding, divider margins |
| sm | 8dp | Compact element spacing, trailing icon inset |
| md | 16dp | Default field padding, card content inset, horizontal screen margin |
| lg | 24dp | Section padding, form group vertical gap |
| xl | 32dp | Layout gaps between major form sections |
| 2xl | 48dp | Major section breaks, top content clearance below app bar |
| 3xl | 64dp | Page-level spacing, bottom safe-area clearance |

**Exceptions:**
- Minimum touch target: 48dp — all tappable elements including show/hide password icon button (REQ-NF from accessibility; Material3 mandate)
- `OutlinedTextField` height: 56dp — Material3 default; do not override
- Bottom of form to primary CTA button: 24dp (lg token)

**Source:** Material3 spacing conventions; 48dp touch target is a Material3 hard minimum.

## Typography

All sizes in `sp` (Android) / `pt` (iOS). Exactly 2 weights used: Regular (400) and SemiBold (600). Exactly 4 sizes used.

| Role | Size | Compose TextStyle | SwiftUI Font | Weight | Line Height |
|------|------|-------------------|--------------|--------|-------------|
| Body | 16sp / 17pt | `MaterialTheme.typography.bodyLarge` | `.body` | 400 (Regular) | 1.5 |
| Label | 14sp / 15pt | `MaterialTheme.typography.bodyMedium` | `.subheadline` | 400 (Regular) | 1.4 |
| Heading | 22sp / 22pt | `MaterialTheme.typography.headlineSmall` | `.title2` | 600 (SemiBold) | 1.2 |
| Display | 28sp / 28pt | `MaterialTheme.typography.headlineMedium` | `.title` | 600 (SemiBold) | 1.15 |

**Usage assignments:**
- Display (28sp): Screen titles (Sign in, Create account, Choose a family)
- Heading (22sp): Section subheadings, Org Picker family name rows
- Body (16sp): Input field labels, body copy, helper text
- Label (14sp): Inline error messages, secondary links, captions

**Source:** Material3 typography scale defaults; consistent with the existing `MaterialTheme` import in `MainActivity.kt`.

## Color

Phase 3 working colors. Phase 4 is the override authority for the final brand palette. Do not hardcode hex values directly in Composables — pass through `MaterialTheme.colorScheme` slots mapped as follows.

| Role | Hex Value | Material3 Slot | SwiftUI Equivalent | Usage |
|------|-----------|----------------|-------------------|-------|
| Dominant (60%) | #FFFFFF | `colorScheme.background` | `Color(.systemBackground)` | Screen background, form background |
| Secondary (30%) | #F4F4F4 | `colorScheme.surfaceVariant` | `Color(.secondarySystemBackground)` | Unfocused input fill, card surface |
| Accent (10%) | #2E7D6F | `colorScheme.primary` | `Color.accentColor` | Primary CTA fill, focused field border, text links |
| Destructive | #B00020 | `colorScheme.error` | `Color(.systemRed)` | Inline field errors, error state icons |

**Accent reserved for:** primary CTA button fill, focused `OutlinedTextField` border ring, text link labels on sign-in/sign-up screens ("Don't have an account?", "Sign in instead"). Accent is NOT used for decorative elements, backgrounds, secondary buttons, or icons outside the above list.

**Dark mode:** Android — rely on Material3's auto dark-mode mapping via `colorScheme`; do not provide a manual dark palette in Phase 3 (Phase 4 responsibility). iOS — use semantic color names (`Color(.systemBackground)`, `Color.accentColor`) which resolve automatically in dark mode.

**Source:** Default working colors. No existing color files found in project. Phase 4 must define the canonical palette before these are used in production-facing screens.

## Screen Inventory

| Screen | Platform | Who Sees It | Entry Condition | States |
|--------|----------|-------------|-----------------|--------|
| Sign In | Android + iOS | Unauthenticated users | App launch, no session | default, loading, error |
| Sign Up | Android + iOS | Unauthenticated users | Tapping "Create one" from Sign In | default, loading, error |

**Scope note (from RESEARCH.md open question 4):** Org creation UI is deferred to Phase 5 (onboarding wizard, REQ-036). The Phase 3 auth flow lands on a minimal stub after org activation — the full 4-tab navigation shell is Phase 5.

**Role gate:** The Invite Caregiver screen is only reachable when `claims.HasRole("org:admin")` is true. Caregiver users who sign in via invitation see the Post-auth Stub directly. Do not show the invite action to caregiver-role users.

## Interaction Contracts

### Form Inputs

- Email field: `keyboardType = Email` (Android: `KeyboardType.Email`; iOS: `.emailAddress`), autocapitalization off, autocorrect off, autocomplete enabled
- Password field: text obscured by default; trailing show/hide toggle (eye icon from Material Icons / SF Symbols); toggle tap target 48dp; `contentDescription` updates between "Show password" and "Hide password"
- IME actions — Sign In: email field "Next" advances to password; password field "Done" submits form
- IME actions — Sign Up: each field "Next" advances to next; last field "Done" submits
- Submit button: disabled while any required field is empty; enters loading state on tap (spinner replaces label text, same button dimensions); re-enabled on error response

### Loading States

- Sign in / sign up / send invitation: primary button replaces label with centered `CircularProgressIndicator` (Compose) or `ProgressView` (SwiftUI); button stays full-width, same height; no skeleton screens
- Org list fetch: full-screen centered spinner while `getOrganizationMemberships()` is in-flight; no skeleton rows

### Navigation Flow

```
[App Launch]
    └── No session → Sign In
                         ├── "Create one" → Sign Up → on success → [membership check]
                         └── on success → [membership check]
                                            ├── 1 org → activate org → Post-auth Stub
                                            └── 2+ orgs → Org Picker → Continue → Post-auth Stub

[Post-auth Stub]
    └── Admin: "Invite" action → Invite Caregiver Screen → success → back to Stub
```

- Back behavior: platform default (Android system back; iOS swipe-back from leading edge); no custom back interception on auth screens
- Auth screens are excluded from the back stack once the user is signed in and an org is active (pop to root, not back to Sign In)

### Keyboard Behavior

- Forms scroll above the software keyboard; use `WindowCompat.setDecorFitsSystemWindows(false)` on Android; iOS uses `.ignoresSafeArea(.keyboard)` on the scroll container
- Tapping outside a text field dismisses the keyboard on both platforms

### Error Display

- Inline errors: appear directly below the triggering field, Label (14sp) typography, Destructive (#B00020) color, accessible live region announcement
- Screen-level errors (network, server): displayed as a text block above the primary CTA, same Label (14sp) / Destructive styling; no modal alerts
- Success state (Invite Caregiver): Material3 `SnackbarHost` (Android) / `.alert` or inline success view (iOS); auto-dismisses after 4 seconds

### Accessibility

- All interactive elements: minimum 48dp touch target enforced
- Error messages: `Modifier.semantics { liveRegion = LiveRegionMode.Polite }` (Compose); `.accessibilityAddTraits(.updatesFrequently)` (SwiftUI)
- Password toggle: `contentDescription` reflects current state ("Show password" / "Hide password")
- Form submission error: focus moves to the first field with an error after submission fails
- Org Picker list: each row is a single focusable element with family name as its accessibility label
- Color contrast: Accent (#2E7D6F) on white (#FFFFFF) = 4.8:1 — passes WCAG AA for normal text and UI components
