<objective>
Bring the iOS app into existence (the `iosApp/` directory currently holds only `.gitkeep`). Create the Xcode project, integrate the ClerkKit SPM package and the shared KMP framework, configure Clerk at launch, implement the shared `AuthRepository` protocol in Swift (so the shared `HttpClient` from 03-02 gets tokens from ClerkKit), and provide a root `ContentView` that routes by auth state.

Purpose: iOS is a first-class target (Phase 3 success criterion 1 requires sign-up/sign-in on both platforms). This plan establishes the iOS foundation that the auth screens (03-06) build on.

Architecture note: Kotlin/Native cannot import the Swift-only ClerkKit package, so the iOS `AuthRepository` implementation lives in Swift and conforms to the SKIE-exported `AuthRepository` protocol from commonMain ŌĆö NOT in `shared/src/iosMain` as RESEARCH/PATTERNS provisionally assumed (Assumption A4 resolved in favor of a Swift implementation). The shared `buildHttpClient(authRepository, baseUrl)` is called from Swift with this implementation injected.

Output: Xcode project; `OneStepTwoApp.swift` (Clerk.configure); `ContentView.swift` (auth routing); Swift `ClerkAuthRepository`; Info.plist key; SPM wiring; a simulator-launchable app.
</objective>
