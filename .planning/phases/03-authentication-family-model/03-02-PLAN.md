<objective>
Stand up the KMP shared HTTP client authentication layer that every platform UI consumes. Add Ktor 3.5.1 client dependencies (core, auth, content-negotiation, kotlinx-json, OkHttp + Darwin engines) to the version catalog and shared module, define the `AuthRepository` interface in commonMain, build the shared `HttpClient` with the Ktor Auth bearer provider configured for race-safe 401 refresh (`nonCancellableRefresh = true`), and implement the Android `AuthRepository` against the Clerk Android SDK.

Purpose: A single shared token-injection layer means Android and iOS share one HTTP client with identical, race-safe bearer auth (REQ-019, REQ-NF-006). The `AuthRepository` interface decouples Ktor from the platform Clerk SDKs.

Output: catalog + Gradle dependency entries; `AuthRepository.kt`; `HttpClientFactory.kt` (bearer plugin, nonCancellableRefresh, host-scoped sendWithoutRequest); Android `ClerkAuthRepository.kt`; a compiling `:shared` Android target.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| shared HttpClient → remote host | Bearer token must only attach to OneStepTwo API hosts |
| Clerk Android SDK → shared module | Session token retrieved in-process; never logged or persisted by the shared layer |
| Gradle build → Maven Central | Ktor + Clerk Android artifacts pulled from Maven Central |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-07 | Information Disclosure | bearer token leakage to third-party hosts | mitigate | `sendWithoutRequest` allowlists only OneStepTwo/localhost/emulator hosts |
| T-3-08 | Denial of Service | concurrent 401 refresh race (KTOR-7852) | mitigate | Ktor `AuthTokenHolder` Mutex + `nonCancellableRefresh = true` (REQ-NF-006) |
| T-3-09 | Spoofing | stale token replayed after refresh | mitigate | `refreshToken()` forces a fresh Clerk token (skip-cache) so 401 retries use a new JWT |
| T-3-SC | Tampering | Ktor + clerk-android-api installs | accept | RESEARCH Package Legitimacy Audit verdict OK/Approved (Maven Central verified, official Clerk + ktorio); no [ASSUMED]/[SUS]/[SLOP] |
</threat_model>
