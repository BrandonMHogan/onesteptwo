<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner → repo | CI executes repo-defined scripts; workflow must not run untrusted codegen or leak secrets in logs |
| CI → external module registries | setup-go and Gradle fetch dependencies during CI |

## STRIDE Threat Register

| Threat ID | Category | Severity | Component | Disposition | Mitigation Plan |
|-----------|----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | medium | CI pipeline (codegen in CI) | mitigate | CI never runs oapi-codegen; it only compiles the committed generated.go (D-03), removing a code-generation step that could alter committed output |
| T-03-02 | Elevation of Privilege | medium | pull_request workflow trigger | mitigate | The workflow only builds and tests; it defines no deploy step and consumes no repository secrets, so a fork PR cannot exfiltrate credentials via CI |
| T-03-03 | Information Disclosure | low | CI logs | accept | Phase 1 CI uses no secrets (debug google-services.json is non-sensitive, committed); nothing secret can leak to logs |
| T-03-SC | Tampering | low | GitHub Actions third-party actions (checkout, setup-go, setup-java, setup-gradle) | mitigate | All actions are official (`actions/*`, `gradle/actions/*`) pinned to major versions; no community/unverified actions used |
</threat_model>
