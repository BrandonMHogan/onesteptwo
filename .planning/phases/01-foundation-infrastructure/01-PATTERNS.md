## Pattern Assignments

### `backend/cmd/server/main.go` (entrypoint, request-response)

**Analog:** RESEARCH.md Pattern 1 — Go stdlib net/http with oapi-codegen

**Core pattern** (from RESEARCH.md §Pattern 1):
```go
package main

import (
    "net/http"
    "os"
    "github.com/BrandonMHogan/onesteptwo/backend/internal/api"
)

func main() {
    port := os.Getenv("PORT")
    if port == "" {
        port = "8080"
    }
    srv := &api.Server{}
    mux := http.NewServeMux()
    api.HandlerFromMux(srv, mux)  // generated registration
    http.ListenAndServe(":"+port, mux)
}
```

**Critical constraints:**
- PORT must come from `os.Getenv("PORT")` — never hardcoded (Railway injects it; Pitfall 4)
- `api.HandlerFromMux` is generated by oapi-codegen — do not hand-write route registration
- No Host header filtering on `/healthz` (Pitfall 5, REQ-001)

### `backend/internal/api/problem.go` (utility, request-response)

**Analog:** RESEARCH.md Code Example — RFC 7807 helper

**Full implementation** (from RESEARCH.md §Code Examples; this is the complete file — 15 lines):
```go
package api

import (
    "encoding/json"
    "net/http"
)

// ProblemDetail is the RFC 7807 error response format.
// The `detail` field is for developers only — never shown to end users.
type ProblemDetail struct {
    Type   string `json:"type"`
    Title  string `json:"title"`
    Status int    `json:"status"`
    Detail string `json:"detail"`
}

// WriteProblem writes an RFC 7807 problem response.
// All Go handlers must use this function for error responses (REQ-NF-001).
func WriteProblem(w http.ResponseWriter, status int, typ, title, detail string) {
    w.Header().Set("Content-Type", "application/problem+json")
    w.WriteHeader(status)
    _ = json.NewEncoder(w).Encode(ProblemDetail{
        Type:   typ,
        Title:  title,
        Status: status,
        Detail: detail,
    })
}
```

**Security constraint:** `detail` must never contain `err.Error()` or stack traces in production (RESEARCH.md §Known Threat Patterns).

```go
import (
    "net/http"
    "net/http/httptest"

    "github.com/BrandonMHogan/onesteptwo/backend/internal/api"

func TestHealthz(t *testing.T) {
    srv := &api.Server{}
    mux := http.NewServeMux()
    api.HandlerFromMux(srv, mux)  // generated by oapi-codegen

    req := httptest.NewRequest(http.MethodGet, "/healthz", nil)
    rec := httptest.NewRecorder()

    if rec.Code != http.StatusOK {
        t.Errorf("expected 200, got %d", rec.Code)
    }
}
```

## Shared Patterns

### RFC 7807 Error Format
**Source:** `backend/internal/api/problem.go` (to be created)
**Apply to:** All future Go handler files
**Pattern:** Call `WriteProblem(w, statusCode, type, title, detail)` — never write raw JSON error responses. `detail` must never contain runtime error strings or stack traces.

### PORT Environment Variable
**Source:** `backend/cmd/server/main.go` (to be created)
**Apply to:** Any Go code that opens a network listener
```go
port := os.Getenv("PORT")
if port == "" {
    port = "8080"
}
```

### Go Module Path
**Apply to:** All Go import paths
**Value:** `github.com/BrandonMHogan/onesteptwo/backend` (locked in PROJECT.md)

### oapi-codegen ServerInterface Pattern
**Apply to:** All future Go API handler files
**Pattern:** New endpoints → add to `api/openapi.yaml` → run `make generate` → implement new method on `api.Server` struct in `server.go`. Never register routes manually in the mux.

### Gradle Version Catalog Reference Pattern
**Apply to:** All `build.gradle.kts` files
**Pattern:** Always use `libs.versions.toml` aliases (`alias(libs.plugins.xyz)`, `implementation(libs.xyz)`) — never hardcode version strings in build files (D-07).

### Java Toolchain
**Apply to:** All Gradle modules that compile Kotlin
```kotlin

    jvmToolchain(21)
}
```
