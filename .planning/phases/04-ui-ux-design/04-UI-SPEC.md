# Phase 4 — UI Design Contract

> Visual and interaction contract for OneStepTwo native mobile (Android + iOS).
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> All D-## references trace to 04-CONTEXT.md locked decisions.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (native mobile — shadcn not applicable) |
| Preset | not applicable |
| Component library (Android) | Jetpack Compose Material3 |
| Component library (iOS) | SwiftUI native |
| Icon library (Android) | Material Icons Extended — outlined style only (Icons.Outlined.*) |
| Icon library (iOS) | SF Symbols — outlined/regular weight only |
| Font (Android) | Roboto (system default, no custom font) |
| Font (iOS) | SF Pro (system default, no custom font) |

Source: D-03, D-05, D-08. No third-party component or icon libraries are permitted in Phase 5+.

## Spacing Scale

Declared values (all multiples of 4):

| Token | dp / pt | Usage |
|-------|---------|-------|
| space-1 | 4 | Icon-to-label gap, tight inline padding, chip internal gap |
| space-2 | 8 | Compact element spacing, field-to-field gap, error message gap |
| space-4 | 16 | Default horizontal screen padding, standard element spacing |
| space-6 | 24 | Section padding, group separation, vertical section gap |
| space-8 | 32 | Layout gap, heading-to-content gap |
| space-12 | 48 | Major section breaks |
| space-16 | 64 | Top/bottom screen breathing room on auth and onboarding screens |

Source: D-26 (8dp base grid). The token naming uses the `space-N` convention where N = dp value / 4.

**Exceptions:**

| Context | Value | Reason |
|---------|-------|--------|
| Minimum tap target (Android) | 48dp height/width | WCAG 2.5.5 / Material3 accessibility |
| Minimum tap target (iOS) | 44pt height/width | Apple HIG |
| Log button height | 52dp / 52pt | Wide pill — comfortably above 48dp minimum |
| Bottom tab bar height (Android) | 56dp + system bottom inset | Material3 NavigationBar spec |
| Status chip height | 28dp / 28pt | Compact pill, not interactive (read-only) |
| Heatmap cell minimum size | 32dp / 32pt | Tappable target (day-detail drill-down) |

## Typography

Six levels per D-25. Weights are limited to two: regular (400) and semibold (600) per minimal palette.

| Role | Size | Weight | Line Height | Letter Spacing | Android TextStyle | iOS Font |
|------|------|--------|-------------|----------------|-------------------|----------|
| Display | 28sp / 28pt | Semibold (600) | 34sp / 34pt (1.21×) | 0sp / 0pt | `headlineMedium` (custom weight) | `.title` + `.semibold` |
| Headline | 24sp / 24pt | Semibold (600) | 30sp / 30pt (1.25×) | 0sp / 0pt | `headlineSmall` (custom weight) | `.system(size: 24, weight: .semibold)` |
| Title | 20sp / 20pt | Semibold (600) | 26sp / 26pt (1.30×) | 0sp / 0pt | `titleLarge` (custom weight) | `.title3` + `.semibold` |
| Body | 16sp / 16pt | Regular (400) | 24sp / 24pt (1.50×) | 0.5sp / 0pt | `bodyLarge` | `.callout` |
| Label | 14sp / 14pt | Regular (400) | 20sp / 20pt (1.43×) | 0.1sp / 0pt | `labelLarge` | `.system(size: 14, weight: .regular)` |
| Caption | 12sp / 12pt | Regular (400) | 16sp / 16pt (1.33×) | 0.4sp / 0pt | `bodySmall` | `.caption` |

**Usage mapping:**

| Level | Used for |
|-------|----------|
| Display (28) | Screen titles (Sign in, Check your email, onboarding step headings, progress tab metric callouts) |
| Headline (24) | Section headings (Settings sections, day-detail date header) |
| Title (20) | Card headers, tab bar labels (active state), bottom sheet titles, child name in home header |
| Body (16) | Standard paragraph text, CTAs/button labels, form field input text, toast message text |
| Label (14) | Secondary text, form field labels, error messages, caregiver name, event type label, date cells |
| Caption (12) | Timestamps, status chip labels, badge unlock dates, heatmap legend labels, "N events pending" |

**Android Theme.kt implementation note:** Phase 5 must create a custom `Typography` object in `ui/theme/Type.kt` that overrides `headlineMedium`, `headlineSmall`, `titleLarge`, `bodyLarge`, `labelLarge`, and `bodySmall` with `fontWeight = FontWeight.SemiBold` (Display, Headline, Title roles) and the line-height / letter-spacing values above.

**iOS implementation note:** Use `.fontWeight(.semibold)` modifier on `.title` and `.title3` preset fonts. For Headline (24pt), use `.system(size: 24, weight: .semibold)`. All font calls use SF Pro by default with no family name specified.

## Color

### 60/30/10 Split

| Role | Light Mode Hex | Dark Mode Hex | Usage |
|------|---------------|---------------|-------|
| Dominant (60%) — background | #FFFFFF | #0E0A14 | Screen backgrounds |
| Secondary (30%) — surface | #FAF5FF | #1A1427 | Cards, nav bar, form fields, tab bar, heatmap empty cells |
| Accent (10%) — primary | #7E22CE | #D8B4FE | See reserved list below |
| Destructive | #B91C1C | #FCA5A5 | Destructive actions only (buttons, error text) |

**Accent (#7E22CE / #D8B4FE) is reserved exclusively for:**
1. Log button background
2. Bottom tab bar active icon + label
3. Primary CTA buttons (onboarding wizard "Continue" / "Start tracking", "Send invite", "Save" in bottom sheet)
4. Progress tab streak number callout
5. Heatmap high-intensity cells (primary-700 / primary-300 in dark mode)
6. Milestone badge unlocked background
7. Focus ring / selection indicator on form fields

Accent is NOT used for: secondary actions, navigation links, general text, dividers, or any ambient decoration.

Source: D-02, D-24.

| Intensity | Light | Dark | Condition |
|-----------|-------|------|-----------|
| Empty | `color.surface-container` | `color.surface-container` | 0 events |
| Low | #F3E8FF (primary-100) | #6B21A8 (primary-800) | 1–2 events |
| Medium | #C084FC (primary-400) | #A855F7 (primary-500) | 3–5 events |

**Android ColorScheme implementation note:** Phase 5 must create `ui/theme/Color.kt` defining the full primary scale as named constants, then `ui/theme/Theme.kt` constructing a `lightColorScheme(...)` and `darkColorScheme(...)` using the semantic tokens above. The `MaterialTheme { }` wrapper in `MainActivity` applies the custom scheme.

**iOS implementation note:** Phase 5 must create an `AppColors` struct (or extension on `Color`) providing all semantic tokens as `static let` constants using `Color(light:dark:)` dynamic provider or `Color("ColorName")` from the asset catalog with appearance variants.

Dark mode follows system preference on first launch; no manual toggle (D-06).

## Motion Tokens

| Token | Value | Android API | SwiftUI API |
|-------|-------|-------------|-------------|
| `motion.duration.short` | 150ms | `tween(150)` | `.easeInOut(duration: 0.15)` |
| `motion.duration.medium` | 300ms | `tween(300)` | `.easeInOut(duration: 0.30)` |
| `motion.duration.long` | 400ms | `tween(400)` | `.easeInOut(duration: 0.40)` |
| `motion.easing.standard` | ease-in-out | `FastOutSlowInEasing` | `.easeInOut` |
| `motion.easing.spring` | spring | `spring(stiffness=300f, dampingRatio=0.6f)` | `.spring(response: 0.35, dampingFraction: 0.7)` |

### Specific Animation Specs

| Interaction | Duration | Easing | Android Compose | SwiftUI |
|-------------|----------|--------|-----------------|---------|
| Tab switch cross-dissolve (D-32) | 120ms | ease-in-out | `AnimatedContent` with `fadeIn(tween(120)) + fadeOut(tween(120))` | `.transition(.opacity)` with `withAnimation(.easeInOut(duration: 0.12))` |
| Log button press scale (D-33) | press: 100ms, release: 200ms | spring | `animateFloatAsState(0.95f, spring(stiffness=500f, dampingRatio=0.7f))` | `.scaleEffect(pressed ? 0.95 : 1.0).animation(.spring(response: 0.10, dampingFraction: 0.7))` |
| Badge glow + scale (D-34) | ~400ms total | spring | `animateFloatAsState(scale 0.7→1.0) + animateFloatAsState(glow alpha 0→0.6→0)` | `withAnimation(.spring(response: 0.4, dampingFraction: 0.6)) { scale = 1.0 }` |
| Toast enter (D-36) | 200ms | ease-out | `AnimatedVisibility(slideInVertically { it } + fadeIn(tween(200)))` | `.transition(.asymmetric(insertion: .move(edge: .bottom).combined(with: .opacity)))` |
| Toast exit (D-36) | 150ms | ease-in | `AnimatedVisibility(slideOutVertically { it } + fadeOut(tween(150)))` | `.transition(.asymmetric(removal: .move(edge: .bottom).combined(with: .opacity)))` |
| Bottom sheet open/close (D-35) | platform default | platform default | `ModalBottomSheet` default (~300ms) | `.sheet` modifier default (natural spring) |

Android animations use **Compose built-in APIs only** — no Lottie, no third-party (D-30).
iOS animations use **SwiftUI built-in APIs only** — no Lottie, no third-party (D-31).

## Component Inventory

Every component must be specced in Phase 4 execution. This section defines the contract for each.

### 1. Bottom Tab Bar

- 4 tabs in order: Home, History, Progress, Settings (REQ-035)
- Android: `NavigationBar` + `NavigationBarItem` (Material3)
- iOS: `TabView` with `.tabItem`
- Active state: `color.primary` icon + label, `radius.pill` active indicator behind icon (Android); `color.primary` icon + label (iOS)
- Inactive state: `color.on-surface` at 60% opacity icon + label
- Hidden during onboarding wizard and full-screen flows (REQ-035)
- Tab bar background: `color.surface`
- No badge indicators in Phase 5 scope

**Tab icon specs:**

| Tab | Android Icon | iOS SF Symbol | Label |
|-----|-------------|---------------|-------|
| Home | `Icons.Outlined.Home` | `house` | "Home" |
| History | `Icons.Outlined.CalendarMonth` | `calendar` | "History" |
| Progress | `Icons.Outlined.TrendingUp` | `chart.line.uptrend.xyaxis` | "Progress" |
| Settings | `Icons.Outlined.Settings` | `gearshape` | "Settings" |

### 2. Log Button

- Shape: wide pill (radius.pill), approximately 70–80% of screen width (D-15)
- Width: `fillMaxWidth(0.75f)` (Android) / `.frame(maxWidth: UIScreen.main.bounds.width * 0.75)` (iOS)
- Height: 52dp / 52pt
- Background: `color.primary`
- Label: "Log", Typography Body (16sp) semibold, `color.on-primary`
- Elevation: overlay level (8dp / shadow radius 8)
- Placement: centered horizontally, directly above tab bar (not overlapping it), 16dp gap from tab bar top
- Haptic: medium impact (`HapticFeedback.mediumImpact`) fires on release (D-12)
- Press animation: scale to 0.95 on press, spring to 1.0 on release (D-33)
- Disabled state: `color.primary` at 38% opacity, label unchanged — only disabled during onboarding wizard
- Accessibility: `contentDescription = "Log potty trip"` (Android) / `.accessibilityLabel("Log potty trip")` (iOS); role = button

### 3. Event Card

Two variants:

**Complete variant** (event_type is set):
- Background: `color.surface`
- Elevation: raised (2dp)
- Corner radius: `radius.md` (12dp)
- Padding: 12dp horizontal, 8dp vertical
- Layout: `[event type icon] [event type label — Label 14sp] [timestamp — Caption 12sp on-surface/60%]`
- Note text: Body 16sp on next line if present, `color.on-surface`
- No caregiver name shown (D-21)

**Pending-details variant** (event_type is null):
- Same as complete but:
- Event type area shows: `[clock icon] "Needs details"` in `color.outline`
- Tapping the card opens the event detail bottom sheet

**Event type icons:**

| Type | Android Icon | iOS SF Symbol |
|------|-------------|---------------|
| pee | `Icons.Outlined.WaterDrop` | `drop` |
| poo | `Icons.Outlined.Circle` (2dp border) | `circle` |
| both | `Icons.Outlined.WaterDrop` (two stacked) | `drop.circle` |
| accident_pee | `Icons.Outlined.Warning` | `exclamationmark.triangle` |
| accident_poo | `Icons.Outlined.Warning` | `exclamationmark.triangle` |
| tried | `Icons.Outlined.RadioButtonUnchecked` | `target` |

Note: accident_pee and accident_poo share the same warning icon; they are differentiated by the label text.

### 4. Heatmap Calendar Cell

- Shape: square with `radius.sm` (8dp) corners
- Minimum size: 32dp × 32dp (tappable; drill-down to day-detail view)
- Intensity states: see Heatmap Intensity Colors table in Color section (D-27)
- Empty cells: `color.surface-container`, no border
- Non-empty cells: filled with intensity color, no border
- Tapping a non-empty cell: navigates to History Day-Detail view
- Tapping an empty cell: no action (ignore tap, no navigation)
- Accessibility: `contentDescription = "[N] events on [date]"` (Android) / `.accessibilityLabel("[N] events on [date]")` (iOS)
- Rolling window: most recent days anchored to today; no fixed-month grid (REQ-033)

### 5. Milestone Badge

**Locked state:**
- Size: 64dp × 64dp circle
- Background: `color.surface-container`
- Icon: relevant outlined icon at 28dp, `color.outline` (muted)
- Label below: Caption (12sp), `color.outline`

**Unlocked state:**
- Background: `color.primary`
- Icon: same icon at 28dp, `color.on-primary`
- Glow ring: semi-transparent `color.primary` at 30% opacity, 4dp inset, animated
- Label below: Caption (12sp), `color.on-background`, bold

**Unlock animation (D-34):**
- Trigger: first render after badge transitions to unlocked
- Scale: 0.7× → 1.0× with `motion.easing.spring` (~400ms)
- Glow overlay: alpha 0 → 0.6 → 0 over 400ms
- Haptic: medium impact fires at start of animation (D-12)
- One-shot: animation only plays once per badge per session

**Four milestone badges (REQ-034):**

| Badge | Icon (Android) | Icon (iOS) | Unlock condition |
|-------|----------------|------------|-----------------|
| First trip | `Icons.Outlined.WaterDrop` | `drop` | First event with type = pee, poo, or both |
| Accident-free day | `Icons.Outlined.CheckCircle` | `checkmark.circle` | First calendar day with zero accident events |
| 7-day streak | `Icons.Outlined.LocalFireDepartment` | `flame` | current streak >= 7 days |
| 30-day streak | `Icons.Outlined.EmojiEvents` | `trophy` | current streak >= 30 days |

### 6. Toast (Post-Log)

- Width: 92% of screen width, centered
- Background: #1C1B1F (near-black, provides contrast in both light and dark mode)
- Corner radius: `radius.lg` (16dp)
- Elevation: overlay (8dp)
- Placement: above tab bar, 8dp gap from tab bar top
- Body text: "Logged. Add a type?" — Body (16sp) in #FFFFFF
- Event type chips: 6 chips in a wrapping row below body text
  - Chip height: 28dp, `radius.pill`, background: `color.secondary`, label: Label (14sp) `color.on-secondary`
  - Tapping a chip: sets event_type, dismisses toast immediately
- Auto-dismiss: 12 seconds (D-17)
- Enter animation: slide up from below + fade in, 200ms (D-36)
- Exit animation: slide down + fade out, 150ms (D-36)
- Accessibility: `LiveRegionMode.Polite` (Android) / `.accessibilityAddTraits(.updatesFrequently)` (iOS)

### 7. Event Detail Bottom Sheet

- Platform: Android `ModalBottomSheet`, iOS `.sheet` (D-11)
- Corner radius: `radius.lg` (16dp) on top corners
- Background: `color.surface`
- Drag handle visible at top center (platform standard)
- Field order (D-18):
  1. Event type selector — segmented/chip row; see chip spec above; pre-selected if set
  2. Text note field — multi-line, Body (16sp), placeholder "Add a note (optional)"
  3. Time adjustment — Label text "Logged at [HH:mm]" with edit icon; opens time picker on tap (optional — available but not required to fill)
  4. Save button — full-width, `color.primary`, Body (16sp) semibold label "Save"
- Sheet title: "Add details" — Title (20sp) semibold
- Dismissal: swipe down or tap outside (platform default); shows confirmation alert if unsaved changes exist

### 8. Status Chips (D-20)

- Two chips shown side-by-side when active; hidden entirely when both counts are zero
- Layout: `Row(horizontalArrangement = Arrangement.spacedBy(8dp))` centered below today's event count
- Each chip:
  - Height: 28dp
  - `radius.pill` corners
  - Background: `color.secondary`
  - Label: Caption (12sp), `color.on-secondary`
  - Elevation: raised (2dp)
- "N need details" chip: tapping opens first incomplete event's detail bottom sheet
- "N pending/syncing" chip: tapping triggers manual sync retry
- Accessibility: each chip has `contentDescription` describing its state

### 9. Child Switcher (D-22)

**Home header:**
- Child name: Title (20sp) semibold, `color.on-background`
- Chevron icon: `Icons.Outlined.ChevronRight` / `chevron.right` SF Symbol, Caption size, `color.outline`
- Combined touch target: full header row, 48dp minimum height
- Chevron and tappable state are hidden when family has exactly one child (REQ-031)

**Child switcher bottom sheet:**
- Same platform sheet as event detail bottom sheet
- Title: "Switch child" — Title (20sp) semibold
- Each child row: Title (20sp) label + checkmark if active, 48dp minimum row height
- No "Add child" action here — child management is in Settings (admin only)
- Accessibility: each row `role = radio`, selected state announced

### 10. Empty State

Consistent pattern across all tabs:

- Layout: centered vertically in available space (not pushing other content)
- Heading: Headline (24sp) semibold, `color.on-background`
- Body: Body (16sp) regular, `color.on-surface` at 70% opacity
- No illustration (D-01 — calm/minimal, text-only empty states)
- No action button in empty state (except Settings empty state: "Invite a caregiver" button)

### 11. Error State

Consistent pattern:

- Layout: centered in available space
- Body: Body (16sp), `color.error`
- "Try again" / "Pull to refresh" copy (see Copywriting Contract)
- Retry action: pull-to-refresh gesture (standard platform)
- Accessibility: `LiveRegionMode.Polite` (Android) / `.accessibilityAddTraits(.updatesFrequently)` (iOS)

### 12. Loading State

- `CircularProgressIndicator` (Android) / `ProgressView()` (iOS)
- Color: `color.primary`
- Centered in available space
- No skeleton loaders in Phase 5 scope
- App startup: full-screen spinner until Clerk.isInitialized (existing pattern from Phase 3)

## Navigation Patterns

### Platform-Specific Back Navigation (D-10)

- iOS: NavigationStack with swipe-from-left-edge gesture (no custom back button)
- Android: system predictive back gesture (no custom back button)
- Both platforms: back pop clears the back stack correctly for pushed detail screens

### Tab Switch (D-32)

- Transition: fade cross-dissolve, 120ms
- Tab state is preserved — switching tabs does not re-create composables / views
- Android: `AnimatedContent` or `NavHost` with custom `ContentTransform`
- iOS: `TabView` default (no custom transition needed; system handles cross-fade)

### Log Button → Toast → Bottom Sheet Flow

1. User taps Log button
2. SQLDelight write fires immediately (no network dependency)
3. Haptic fires on release; button scales 0.95 → 1.0 (spring, 200ms release)
4. Toast slides up from above tab bar (200ms enter)
5. User taps event type chip in toast → event_type written to SQLDelight → toast exits (150ms)
6. User taps "add details" in toast (or taps pending-details card later) → bottom sheet opens (platform default timing)
7. User fills fields and taps Save → sheet dismisses, event updated in SQLDelight

### History Drill-Down

- Heatmap cell tap → push History Day-Detail screen onto NavigationStack / NavigationController
- Tab bar hides during day-detail view
- Back navigation returns to History tab with heatmap visible

### Onboarding Wizard Navigation

- Steps are push navigation within a single NavigationStack (not a tab-based flow)
- No back navigation allowed from consent step (step 4) once shown — wizard is linear
- Step 5 (completion) → replace the navigation stack root with the main app (Tab bar shell)

### Auth → Main App Transition

After sign-in/wizard completion:
- Replace navigation stack root with MainAppShell (Tab bar)
- Back stack cleared — user cannot navigate back to auth screens
- Matches existing Phase 3 `popUpTo(0) { inclusive = true }` pattern (AppNavigation.kt)

## Platform-Specific Exceptions (D-08)

| Feature | Unified spec | Android exception | iOS exception |
|---------|-------------|-------------------|---------------|
| Back navigation | Swipe / gesture | System predictive back gesture | NavigationStack swipe-from-left-edge |
| Event detail sheet | Bottom sheet | `ModalBottomSheet` | `.sheet` modifier card style, swipe to dismiss |
| Bottom tab bar | 4-tab bottom anchored | `NavigationBar` Material3 | `TabView` with `.tabItem` |
| Progress indicator | Circular spinner | `CircularProgressIndicator` | `ProgressView()` |
| Haptic | Defined interactions | `HapticFeedback.performHapticFeedback(HapticFeedbackType....)` | `UIImpactFeedbackGenerator(style: .medium)` |
| Alert / confirm dialog | Native platform alert | `AlertDialog` (Compose) | `.alert` modifier (SwiftUI) |
